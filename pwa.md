## Purpose

Standards for building Progressive Web Apps (PWAs) targeting mobile home screen install, primarily iPhone (iOS Safari).

Applies to any React/Vite project that uses `vite-plugin-pwa` and is intended to be installed as a home screen app.

---

## Stack

| Layer | Choice |
|---|---|
| Build | Vite + `vite-plugin-pwa` |
| Manifest | Configured in `vite.config.ts` via `VitePWA()` |
| Service worker | `registerType: 'autoUpdate'` (generated by Workbox) |
| iOS meta tags | Manually in `index.html` (Safari ignores the web manifest for install) |

---

## Vite Config

```ts
import { VitePWA } from 'vite-plugin-pwa';

VitePWA({
  registerType: 'autoUpdate',
  manifest: {
    name: 'App Name',
    short_name: 'App',
    start_url: '/your-base-path/',
    display: 'standalone',
    background_color: '#ffffff',
    theme_color: '#e63946',
    icons: [
      { src: '/icons/apple-touch-icon.png', sizes: '180x180', type: 'image/png' },
    ],
  },
})
```

- `display: 'standalone'` is required for the app to run without browser chrome.
- `start_url` must match `base` in `vite.config.ts` — use the deployed sub-path (e.g. `/my-app/` for GitHub Pages).
- `theme_color` sets the status bar tint on Android.

## `index.html` — iOS Meta Tags

Safari does not use the web manifest for PWA install. These meta tags are required:

```html
<!-- Link the manifest for Android and other browsers -->
<link rel="manifest" href="/your-base-path/manifest.webmanifest" />

<!-- iOS home screen install -->
<link rel="apple-touch-icon" sizes="180x180" href="/icons/apple-touch-icon.png" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-title" content="App Name" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />

<!-- Safe area support for notch / Dynamic Island / home indicator -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />

<!-- Android status bar colour -->
<meta name="theme-color" content="#e63946" />
```

`viewport-fit=cover` is mandatory — without it `env(safe-area-inset-*)` returns zero on notched iPhones.

## Standalone Gate

A PWA intended only for home screen use should detect whether it is running installed or in the browser and show an install instruction screen when it is not installed.

```ts
function isRunningStandalone(): boolean {
  return (
    window.matchMedia('(display-mode: standalone)').matches ||
    (window.navigator as Navigator & { standalone?: boolean }).standalone === true
  );
}
```

- The `matchMedia` check covers Android and desktop PWAs.
- `navigator.standalone` is iOS Safari-specific.

### Dev bypass

The gate blocks development in a regular browser. Bypass it in dev mode:

```tsx
// App.tsx
if (!import.meta.env.DEV && !isRunningStandalone()) {
  return <GateScreen />;
}
return <Layout />;
```

Never commit with the gate permanently disabled — the check must be present in production builds.

## Safe Area CSS

iPhones with a notch or Dynamic Island (iPhone X and later) have hardware insets at the top and bottom. Use CSS environment variables to clear these areas.

```css
/* Top: Dynamic Island / notch */
padding-top: env(safe-area-inset-top);

/* Bottom: home indicator */
padding-bottom: env(safe-area-inset-bottom);
```

In Tailwind:
```html
<!-- Bottom nav clears home indicator -->
<nav class="pb-[env(safe-area-inset-bottom)]">

<!-- Content area avoids the notch -->
<div class="pt-[env(safe-area-inset-top)]">
```

These values are `0` on non-notched devices — safe to apply unconditionally.

## PWA Icon

iOS applies its own rounded-corner mask to the `apple-touch-icon`. Supply a square 180×180px PNG with no rounding:

```
public/
└── icons/
    └── apple-touch-icon.png   (180×180, square corners)
```

The icon path must match exactly what is in the `<link rel="apple-touch-icon">` tag and the manifest `icons` array.

## Deployment and Base Path

When deploying to a sub-path (e.g. GitHub Pages at `/repo-name/`), every path reference must use the same base:

```ts
// vite.config.ts
export default defineConfig({
  base: '/repo-name/',
  // ...
});
```

```html
<!-- index.html — manifest href must include base -->
<link rel="manifest" href="/repo-name/manifest.webmanifest" />
```

The `start_url` in the manifest must also match. A mismatch causes the service worker to fail silently or the install prompt to never appear.

## Environment Variables (Vite)

Vite bakes `VITE_*` environment variables into the static bundle at build time. There is no runtime environment injection on GitHub Pages.

- Set credentials in `.env.local` (gitignored) for local development.
- For production, rebuild and redeploy after changing credentials — the new values are baked into the new bundle.
- Never rely on server-side environment injection for a static PWA.
