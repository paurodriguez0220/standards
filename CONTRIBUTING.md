# Contributing

These are living documents. Update them when a standard changes based on experience, when you discover a gap, or when a tool or practice becomes obsolete.

## How to contribute

1. Branch from `main`
2. Edit the relevant `.md` file
3. If adding a new document, add it to the index table in `README.md`
4. PR back to `main` with a clear description of what changed and why

## Writing style

- Direct and opinionated. State the rule, then the reason.
- Use examples where the rule is non-obvious.
- No jargon unless it's defined or universally understood.
- Prefer short bullet lists and code blocks over long paragraphs.
- Every document should answer: *what do I do, and why?*
